Name Type Default Required Description
data array[json object] undefined O

N.list에 바인딩 할 데이터를 지정 합니다.

Specifies the data to bind to N.list.

N.list의 context 요소는 반드시 ul 과 li 태그로 작성 해야 합니다.

The context element of N.list must be written with ul and li tags.

행 데이터의 프로퍼티명과 요소의 id 속성값이 같으면 데이터가 바인딩 됩니다.

If the property name of the row data and the value of the id attribute of the element are the same, the data is bound.

Natural-JS 의 모든 데이터 관련 컴포넌트는 빠른 바인딩 속도를 위해 어쩔수 없이 id 속성을 사용 합니다.

All data-related components of Natural-JS inevitably use id attribute for fast binding speed.

<ul class="list-context">
    <li>
        <input id="name" type="text">
        <input id="age" type="text">
        <input id="email" type="text">
        <span id="registered"></span>
    </li>
</ul>

height number 0 X

리스트 body의 높이를 지정 합니다.

Specifies the height of the list body.

설정한 값이 0보다 크면 리스트 바디에 스크롤바가 생기면서 설정한 높이로 고정 됩니다. 0으로 설정하면 모든 데이터를 그리드 바디에 보여 줍니다.

If the set value is greater than 0, a scroll bar is created on the list body, and it is fixed at the set height. If set to 0, all data is shown in the list body.

Pagination(N.pagination) 컴포넌트와 결합하여 페이징 된 데이터를 N.list에 바인딩 할 때는 0으로 설정하여 모든 데이터를 한번에 표시 하는것을 권장 합니다.

When binding paged data to the N.list in combination with the Pagination(N.pagination) component, it is recommended to display all data at once by setting it to 0.

If set to false, validation is not performed on the input value when the input element is focus-out.

If set to true, row elements and row data created to the top of the list when the add method is called.

addTop 옵션을 false 설정 하면 데이터 동기화 문제가 발생하여 scrollPaging.size 와 createRowDelay 옵션의 설정 값이 0으로 강제 설정 됩니다. 이로인해 데이터 바인딩 성능이 저하 될 수 있습니다.

Setting the addTop option to false causes data synchronization issues, which forces the setting values ​​of scrollPaging.size and createRowDelay options to zero. This can degrade the performance of data binding.

If set to true, height of the grid body can be resized.
windowScrollLock boolean true X

true로 설정 하면 데이터 목록 요소 위에서 마우스 휠로 스크롤 할 때 브라우저의 윈도우 스크롤을 비활성화 합니다.

If set to true, disables browser window scrolling when scrolling with the mouse wheel over data list elements.

데이터 목록 요소가 처음이나 마지막으로 스크롤 되었을 때 브라우저 윈도우 스크롤이 위나 아래로 스크롤되는 브라우저의 기본 동작을 차단 합니다.

Blocks the browser's default behavior of scrolling the browser window up or down when the data list element is scrolled first or last.

true로 설정 하면 revert 기능이 활성 화 되고 revert 메서드를 사용 할 수 있습니다.

If set to true, the revert function is enabled and the revert method can be used.

revert 기능이 활성화 되면 최초 바인딩된 데이터가 메모리에 적제되므로 메모리 사용량이 증가 합니다.

When the revert function is activated, the memory usage increases because the first bound data is loaded into the memory.
select boolean false X

true로 설정 하면 행을 선택(단일 행 선택) 했을 때 onSelect 이벤트가 발생 하고 행 요소(tbody)의 class 속성에 list_selected__ 값이 토글 됩니다.

If set to true, the onSelect event fires when a row is selected(Single row selection), and the list_selected__ value is toggled in the class attribute of the row element(tbody).

false로 설정 하면 select 옵션이 true 일때 선택 된 행을 다시 선택해도 선택이 취소 되지 않습니다.

If set to false, when the select option is true, selecting the selected row again does not cancel the selection.

multiselect 옵션이 true 일 경우에는 unselect 옵션이 적용 되지 않습니다.

If the multiselect option is true, the unselect option has no apply.

id 속성을 지정하여 데이터를 바인드 하면 정상적으로 작동하지 않습니다. 선택 된 행의 인덱스를 가져오는 용도로만 사용 바랍니다.

scrollPaging.size

scrollPagingSize

 number 100 X

스크롤 페이징 할때 한번에 바인딩 할 행 수를 지정 합니다.

Specifies the number of rows to bind at a time when scroll paging.

N.list 는 스크롤 페이징 기능이 기본적으로 활성화 되며 0 을 지정하면 스크롤 페이징 기능이 비 활성화 됩니다.

In N.list, the scroll paging feature is enabled by default, if 0 is specified, the scroll paging feature is disabled.

size 옵션 값은 다음과 같이 scrollPaging 옵션 객체의 하위 속성으로 지정해야합니다.

The size option value must be specified as a sub-property of the scrollPaging option object as follows.

...
    scrollPaging : {
        size : 50
    }
...

너무 작게 지정하면 스크롤이 생성 되지 않아 데이터가 모두 표시 되지 않습니다. 반드시 행 들이 리스트 바디를 넘칠 수 있는 양을 지정 해 주어야 합니다.

If it is set too small, scrolling is not created and all data is not displayed. You must specify the amount by which rows can overflow the list body.

너무 크게 지정하면 스크롤 페이징이 작동될때 브라우저에 부하가 발생되므로 리스트에 입력요소나 이미지가 들어 있다면 100 이하로 설정하고 그렇지 않다면 1000 이하로 설정하는 것을 권장합니다.

If set too large, browser performance is overhead when scroll paging is enabled, so it is recommended to set it to 100 or less if the input element or image is included in the list, otherwise 1000 or less.

    // this : N.list 인스턴스
    // rowIdx : 선택된 행의 index
    // rowEle : 선택된 행의 row element(li)

    // beforeRowIdx : 이전 선택된 행의 index

    // isLastPage : 스크롤 페이징 시 마지막 페이지 여부(스크롤 페이징 하지 않으면 true 반환)
