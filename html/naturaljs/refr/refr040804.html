Name Type Default Required Description

Specifies the data to bind to N.list.
context jquery object|jquery selector string null O

N.list를 적용 할 context 요소를 지정 합니다.

Specifies the context element to apply N.list.
height number 0 X

리스트 바디(body)의 높이를 지정 합니다.

입력한 값이 0보다 크면 리스트의 높이를 입력한 값으로 고정하고 스크롤을 표시 해 줍니다. 0을 입력하면 스크롤 없이 전체 데이터를 표시 해 줍니다.
validate boolean true X

false로 설정 하면 입력 요소에서 focus-out 되었을 때 입력 값에 대한 유효성 검증을 실행 하지 않습니다.

If set to false, validation is not performed on the input value when the input element is focus-out.
html boolean false X

true로 설정 하면 data 를 bind 할 때 데이터의 HTML 이 적용 됩니다.

If set to true, HTML of data is applied when binding data.
addTop boolean false X

add 메서드를 호출했을 때 새로운 행요소와 데이터세트를 리스트의 상단에 추가할지 하단에 추가할지를 결정 합니다.

true 이면 상단에 추가하고 false 이면 하단에 추가 합니다.

addTop 옵션이 false 이면 scrollPaging.size 옵션과 createRowDelay 옵션을 강제로 0으로 설정 하여 스크롤페이징 기능과 행이 순서대로 그려지는 효과를 비활성화 시킵니다.
vResizable boolean false X

true로 설정 하면 그리드 바디의 높이를 조절 할 수 있습니다.

If set to true, height of the grid body can be resized.
windowScrollLock boolean true X

리스트 바디 영역에 mouseover 시 윈도우 스크롤을 비활설화 할 것인지 여부

리스트안에서 마우스 스크롤 시 리스트의 끝 영역이나 시작영역에 가면 브라우저 윈도우 스크롤이 자동으로 올라가거나 내려가서 리스트가 표시된 부분이 넘어가게 됩니다. 때문에 리스트 바디영역에 마우스가 올라가면 윈도우 스크롤을 비활성 시킵니다. false로 지정하면 브라우저 기본모드로 동작 됩니다.
revert boolean false X

revert 기능을 활성(true), 비활성(false) 시킵니다.

revert 기능을 활성화 하면 최초 바인딩된 데이터를 메모리에 적제 하므로 데이터 바인딩 성능이 저하될 수 있습니다.
select boolean false X

true로 설정 하면 행을 선택(단일 행 선택) 했을 때 onSelect 이벤트가 발생 하고 행 요소(tbody)의 class 속성에 list_selected__ 값이 토글 됩니다.

If set to true, the onSelect event fires when a row is selected(Single row selection), and the list_selected__ value is toggled in the class attribute of the row element(tbody).
multiselect boolean false X

true로 설정 하면 행을 선택(다중 행 선택) 했을 때 onSelect 이벤트가 발생 하고 행 요소(tbody)의 class 속성에 list_selected__ 값이 토글 됩니다.

If set to true, the onSelect event fires when a row is selected(Multiple row selection), and the list_selected__ value is toggled in the class attribute of the row element(tbody).
unselect boolean true X

false로 설정 하면 select 옵션이 true 일때 선택 된 행을 다시 선택해도 선택이 취소 되지 않습니다.

If set to false, when the select option is true, selecting the selected row again does not cancel the selection.

multiselect 옵션이 true 일 경우에는 unselect 옵션이 적용 되지 않습니다.

If the multiselect option is true, the unselect option has no apply.
checkAll jquery object null X

리스트의 행을 선택하기 위한 체크박스를 모두 선택하는 checkbox input 요소 지정 합니다.

N.grid 와 달리 문자열로 지정하지 않고 정확하게 해당 요소를 Select 하여 jquery object 타입으로 지정 바랍니다.

전체 선택 만 해 주고 체크박스의 클릭 이벤트는 발생 시키지 않습니다.
checkAllTarget string null X

리스트의 행을 선택하기 위한 체크박스 요소들을 지정(멀티 선택) 합니다.

jQuery selector 문자열로 지정하며 selector 의 context 는 리스트의 행 요소(li)로 자동 지정 됩니다.

checkAllTarget 대상 요소는 반드시 li 영역에 있어야 합니다.

checkAllTarget 옵션과 checkSingleTarget 옵션은 둘중 하나만 사용 할 수 있습니다.

id 속성을 지정하여 데이터를 바인드 하면 정상적으로 작동하지 않습니다. 선택 된 행의 인덱스를 가져오는 용도로만 사용 바랍니다.
checkSingleTarget string null X

리스트의 행을 선택하기 위한 체크박스 요소들을 지정(단일 선택) 합니다.

jQuery selector 문자열로 지정하며 selector 의 context 는 리스트의 행 요소(li)로 자동 지정 됩니다.

checkSingleTarget 대상 요소는 반드시 li 영역에 있어야 합니다.

checkAllTarget 옵션과 checkSingleTarget 옵션은 둘중 하나만 사용 할 수 있습니다.

id 속성을 지정하여 데이터를 바인드 하면 정상적으로 작동하지 않습니다. 선택 된 행의 인덱스를 가져오는 용도로만 사용 바랍니다.
hover boolean false X

리스트 행들의 HOVER 효과 적용 여부

리스트 행에 mouseover 시 해당 행의 LI요소에 "list_hover__" 스타일클래스가 생성되고 선택이 해제되면 생성된 스타일클래스가 제거 됩니다.
createRowDelay number(ms) 1 X

행 요소들이 생성될 때마다의 딜레이를 지정 할수 있습니다.

0 지정시 모든행을 한번에 만들어 표현 합니다. 이렇게 되면 브라우저는 모든행이 생성될때까지 멈춰있게 됩니다.
scrollPaging.size

scrollPagingSize

 number 100 X

스크롤 페이징 할때 한번에 바인드하는 행수를 지정 합니다.

Specifies the number of rows to bind at a time when scroll paging.

N.list 는 스크롤 페이징 기능이 기본적으로 활성화 되며 0 을 지정하면 스크롤 페이징 기능이 비 활성화 됩니다.

In N.list, the scroll paging feature is enabled by default, if 0 is specified, the scroll paging feature is disabled.

size 옵션 값은 다음과 같이 scrollPaging 옵션 객체의 하위 속성으로 지정해야합니다.

The size option value must be specified as a sub-property of the scrollPaging option object as follows.

...
    scrollPaging : {
        size : 50
    }
...

너무 작게 지정하면 스크롤이 생성 되지 않아 데이터가 모두 표시 되지 않습니다. 반드시 행 들이 리스트 바디를 넘칠 수 있는 양을 지정 해 주어야 합니다.

If it is set too small, scrolling is not created and all data is not displayed. You must specify the amount by which rows can overflow the list body.

너무 크게 지정하면 스크롤 페이징이 작동될때 브라우저에 부하가 발생되므로 리스트에 입력요소나 이미지가 들어 있다면 100 이하로 설정하고 그렇지 않다면 1000 이하로 설정하는 것을 권장합니다.

If set too large, browser performance is overhead when scroll paging is enabled, so it is recommended to set it to 100 or less if the input element or image is included in the list, otherwise 1000 or less.

rowHandlerBeforeBind function null X

리스트에서 data를 bind 하거나 add 할때 행이 렌더링 될 때 마다 발생 하는 이벤트 입니다. 이벤트 콜백 메서드의 인자로 생성될 당시의 row index, 행 요소, 로우 데이터를 넣어줍니다.

rowHandlerBeforeBind : function(rowIdx, rowEle, rowData) {
	// this : N.list 인스턴스
	// rowIdx : 생성되는 행의 index
	// rowEle : 생성되는 행의 row element(tbody)
	// rowData : 생성되는 행의 row data
}

rowHandlerBeforeBind 는 행 마다 값이 바인드 되기 전 실행 됩니다.
rowHandler function null X

리스트에서 data를 bind 하거나 add 할때 행이 렌더링 될 때 마다 발생 하는 이벤트 입니다. 이벤트 콜백 메서드의 인자로 생성될 당시의 row index, 행 요소, 로우 데이터를 넣어줍니다.

rowHandler : function(rowIdx, rowEle, rowData) {
	// this : N.list 인스턴스
	// rowIdx : 생성되는 행의 index
	// rowEle : 생성되는 행의 row element(li)
	// rowData : 생성되는 행의 row data
}

rowHandler 는 행 마다 값이 바인드 된 후 실행 됩니다.
onSelect function null X

리스트의 행이 선택될때마다 발생되는 이벤트이다. 이벤트 콜백 메서드의 인자로 선택한 행의 row index, 행 요소, 로우 데이터, 바로전에 선택한 row index를 넣어준다.

onSelect : function(rowIdx, rowEle, rowData, beforeRowIdx, e) {
	// this : N.list 인스턴스
	// rowIdx : 선택된 행의 index
	// rowEle : 선택된 행의 row element(li)
	// rowData : 선택된 행의 row data
	// beforeRowIdx : 이전 선택된 행의 index
	// e : Click event object
}

select 옵션이 true 일 때만 발생 됩니다.

unselect 옵션이 false(기본값) 이면 선택이 해제 되었을 때 rowIdx 인자 값에 -1을 반환 하고 선택 되었을 때는 선택 된 행의 인덱스를 반환 합니다. unselect 옵션이 true 이면 선택 해제 기능이 비활성화 되기 때문에 rowIdx 값은 항상 선택 된 행의 인덱스를 반환 합니다.

onBind function null X

데이터가 바인드 될 때마다 발생되는 이벤트입니다. 이벤트 핸들러 함수의 인자로 context, data, isFirstPage, isLastPage 가 반환 됩니다.

onBind : function(context, data, isFirstPage, isLastPage) {
	// this : N.list 인스턴스
	// context : context(리스트 컨텍스트 요소)
	// data : 리스트에 바인드 된 데이터
    // isFirstPage : 스크롤 페이징 시 첫번째 페이지 여부(스크롤 페이징 하지 않으면 true 반환)
    // isLastPage : 스크롤 페이징 시 마지막 페이지 여부(스크롤 페이징 하지 않으면 true 반환)
}

스크롤페이징에 의해 페이징 될 때도 해당 행수 만큼 페이지가 완성 될 때마다 발생합니다.
