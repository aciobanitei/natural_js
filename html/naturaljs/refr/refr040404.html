팝업의 넓이

  • number 타입 일 경우 지정한 수치만큼(px) 넓이가 지정 됩니다.
  • function 일 경우 msgContext(modal 옵션이 true 일때 바닥에 깔리는 요소), msgContents(팝업 컨텐츠 요소) 가 인자로 전달 되고 return 된 값으로 넓이가 지정 됩니다. 
    ...
// 대화상자의 넓이를 화면에 꽉 채우기

    return $(window).width();
},
...

onOpen string null X

팝업이 열릴 때마다 발생되는 이벤트

문자열로 함수명칭만 정의 하고 실제 이벤트 핸들러 메서드는 불러오는 팝업 페이지의 Controller 에 구현되어 있어야 합니다.

open 함수를 실행 할 때 첫번째 인자로 지정한 오브젝트가 onOpen 이벤트 핸들러 메서드의 첫번째 인자(onOpenData)로 반환 됩니다.

url 옵션을 지정하여 다른 파일의 컨텐츠를 팝업으로 생성 했을 때만 사용가능 합니다.

팝업 컨텐츠가 처음 로딩 될 때는 N.cont 오브젝트의 init 함수가 실행 된 다음 onOpen 함수가 실행 됩니다.
onOpenData object null X

onOpen 이벤트의 첫번째 인자로 지정되어 팝업에 전달될 데이터

url을 지정하여 다른 페이지를 가져왔을때만 사용가능 합니다.

open 함수의 첫번째 argument 로 넣어줘도 됩니다(동적인 데이터 전달 시 사용).
onClose function null X

팝업이 닫힐 때마다 발생되는 이벤트.

onClose : function(onCloseData) {
    // this : N.popup 인스턴스
    // onCloseData : 팝업에서 전달 해준 데이터
}

팝업 컨텐츠 로드가 완료 되면 팝업의 Controller object 에 caller 라는 속성을 넣어 주는데 이 caller 는 부모페이지에서 정의 한 N.popup 인스턴스 입니다.
열린 팝업 자신이 자신을 닫을 때는 팝업 페이지의 Controller(N.cont) object 에서 "this.caller.close(onCloseData);" 명령을 실행 하면 됩니다.
위 명령을 실행하면 팝업이 닫히면서 onClose 이벤트 핸들러 메서드의 인자로 onCloseData 가 전달 됩니다.

url 옵션을 지정하여 다른 파일의 컨텐츠를 팝업으로 생성 했을 때만 사용가능 합니다.
onCloseData object null X

onClose 이벤트의 첫번째 인자로 지정되어 팝업의 부모페이지에 전달될 데이터

url을 지정하여 다른 페이지를 가져왔을때만 사용가능 합니다.

팝업에서 this.caller.close 함수의 첫번째 argument 로 넣어줘도 됩니다(동적인 데이터 전달 시 사용).
