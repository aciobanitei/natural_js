N.comm N/A N/A N.comm

N.comm 인스턴스를 생성합니다.

Create an N.comm instance.

N.comm 은 new 연산자로 인스턴스를 생성하지 않고 N.comm 객체를 바로 사용 할 수 있습니다.

N.comm 객체에서 submit 함수를 호출 하면 Communicator.request 객체 인스턴스가 요청마다 생성 됩니다.

var comm = N.comm(param|element|url, opts|url);
param|element|url jquery object[array[object]]|jquery object[object]|jquery object[element]|object|string N/A

첫번째 인자에 object 나 array[object] 타입의 객체가 들어있는 jquery object 로 설정 하고 두번째 인자(url) 에 JSON 데이터를 반환하는 서버 URL 을 설정 하면 설정 된 객체를 문자열로 변환 한 후에 서버에 GET 이나 POST 파라미터로 전달 합니다. 이때 이 인자값은 Communicator.request 의 기본 옵션의 data 속성 값으로 설정 됩니다.

N.comm({ "param" : "value" }, "data.json").submit(function(data) {});

type(HTTP METHOD) 옵션이 GET 일 경우 data 속성 값(param)이 object 이거나 array[object] 타입이면 q 라는 파라미터 키에 URL encoding 된 파라미터 문자열이 값으로 설정 됩니다.
  ex) data.json?q=%5B%7B%22param%22%3A%22value%22%7D%5D // q=[{"param":"value"}]
GET 파라미터는 최대 전송 할 수 있는 양이 정해져 있기 때문에 브라우저별 GET 파라미터의 최대 전송량을 파악하여 사용하기 바랍니다.

첫번째 인자에 HTML 요소가 담겨있는 jquery object 타입을 설정 하고 두번째 인자(url) 에 HTML 페이지를 반환하는 서버 URL 을 설정 하면 지정한 요소에 불러온 페이지를 추가 해 줍니다.

페이지에 파라미터를 전달하는 경우에는 Communicator.request 객체의 attr 메서드를 사용 합니다.

N.comm(N("#contents"), "page.html").submit();

// 페이지 파라미터 전송
// 불러오는 페이지의 Controller object 의 init 함수의 request 인자(arguments[1])로 받을 수 있습니다.
N.comm(N("#contents"), "page.html").request.attr("pageParam", "value").submit();

object 타입을 설정 하면 Communicator.request 객체의의 기본 옵션으로 지정 됩니다.

N.comm({ "param" : "value" }, {
    url : "data.json"
}).submit(function(data) {});

string 타입을 설정 하면 Communicator.request 객체의 기본 옵션의 url 옵션으로 설정 됩니다.

N.comm("data.json").submit(function(data) {});
opts|url object|string N/A

object 타입을 설정 하면 Communicator.request 객체의 기본 옵션으로 설정 됩니다.

string 타입을 설정 하면 Communicator.request 객체의 기본 옵션의 url옵션으로 설정 됩니다.
N(obj).comm N/A N/A N.comm

N.comm의 객체 인스턴스를 jQuery 플러그인 방식으로 생성합니다.

Create an object instance of N.comm with jQuery plugin method.

var comm = N(param|element).comm(opts|url);

객체 인스턴스를 생성하는 방식은 다르지만 "new N.comm()" 로 생성한 인스턴스와 "N().comm" 로 생성한 인스턴스는 동일 합니다. N() 함수의 첫번째 인자가 new N.comm 생성자의 첫 번째 인자로 설정 됩니다.

The method of creating object instances is different, but the instance created with "new N.comm()" and the instance created with "N().comm" are the same. The first argument of the N () function is set as the first argument of the new N.comm constructor.

N() 함수의 인자 타입 중에 string 타입은 N.comm 생성자의 첫 번째 인자(url)로 설정 되지 않습니다.

N() 함수의 인자 타입 중에 string 타입은 N.comm 생성자의 첫 번째 인자(url)로 설정 되지 않습니다.
opts|url object|string N/A

N.comm 함수의 두번째 인자(opts|url)와 같습니다.

Same as the second argument(opts|url) of the N.comm function.
