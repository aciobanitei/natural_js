Name Arguments Type Return Description
N.data.refine N/A N/A array[json object] 서버나 JSON 데이터가 담긴 파일에서 가져온 JSON 데이터를 Natural-JS에서 사용할수 있는 타입의 json 타입으로 만들어 줍니다.
obj array[json object] N/A 정제 할 데이터
listId string N/A 서버나 JSON 데이터가 담긴 파일에서 여러개의 리스트를 반환했을경우의 해당 리스트의 ID 값
N.data.filter N/A N/A array[json object] 지정한 조건으로 array[json object] 에서 또다른 array[json object] 데이터를 추출 합니다.
arr array[json object] N/A 필터링 할 데이터
condition function|string N/A

필터링 조건을 입력 합니다.

인자로 콜백 펑션을 입력 하면 콜백에서 true 를 반환한 행들 만 반환 합니다.

var fData = N.data.filter([
	{ name : "홍길동", age : 18 },
	{ name : "Mike", age : 16 },
	{ name : "Mike", age : 14 }
], function(item) {
    return item.name === "Mike" && age === 16;
});

/*
fData의 값(결과값) :
[
	{ name : "Mike", age : 16 }
]
*/

인자로 조건 문자열을 입력 하면 조건에 맞는 행들 만 반환 합니다.

var fData = N.data.filter([
	{ name : "홍길동", age : 18 },
	{ name : "Mike", age : 16 },
	{ name : "Mike", age : 14 }
], 'name === "Mike"');

/*
fData의 값(결과값) :
[
	{ name : "Mike", age : 16 },
	{ name : "Mike", age : 14 }
]
*/

필터링 속도는 문자열로 조건을 입력하는 것 보다 콜백으로 조건을 입력하는 방식이 더 빠릅니다.

(추후 지원 예정)문자열로 조건을 입력하는 방식은 and(&&) 나 or(||) 연산식을 지원하지 않고 단일 조건식만 지원 합니다.

$([ { col01 : 1 }, { col01 : 2 } ]).datafilter("col01 === 1") 와 같이 jQuery object 타입의 배열은 datafilter 명령어로 사용하면 됩니다.

N(obj).datarefine N/A N/A jquery object[json object]

N.data.refine을 플러그인 방식으로 초기화 하는 방식입니다.

N.data.refine과 동작되는 로직은 같으며 실제 N() 의 인자로 들어온 값이 N.data.refine()의 첫 번째 인자로 지정 됩니다.
listId string N/A N.data.refine의 listId 인자 와 같습니다.

condition function|string N/A N.data.filter의 condition 인자 와 같습니다.
