기본옵션 Default Options

Name Type Default Required Description
context jquery object|jquery selector string null O

N.docs를 적용 할 context 요소를 지정 합니다.

Specify the context element to apply N.docs.
multi boolean true X

true로 설정하면 탭 기반 Multiple-Document Interface(MDI) 형태로 N.docs가 생성되고 false로 설정하면 일반적인 Single-Document Interface(SDI) 형태로 N.docs가 생성됩니다.

If set to true, N.docs are generated as a tab-based Multiple Document Interface(MDI) type, and when set to false, N.docs are generated as a generic Single Document Interface(SDI) type.

  • true : 탭과 페이지 컨텐츠들을 표시 합니다.
  • true : Displays tabs and page contents.
  • false : 탭은 표시 하지 않고 단 한 개의 페이지 컨텐츠만 표시 합니다.
  • false : The tab is not displayed, and only one page content is displayed.
maxStateful number 0 X

multi 옵션이 true 인 경우 추가 탭 컨텐츠를 열 때마다 웹 브라우저가 느려지지 않도록 최대 상태 유지 탭 컨텐츠 수를 설정할 수 있습니다.

If the multi option is true, the maximum number of stateful tab contents can be set to prevent the web browser from slowing down whenever additional tab contents are opened.

새로운 컨텐츠를 열때 열려있는 탭 컨텐츠의 개수가 최대 상태유지 개수를 초과 하면 제일 먼저 열린 탭 컨텐츠의 상태가 제거되고 상태유지가 제거 된 탭을 클릭 하면 해당 컨텐츠가 다시 로딩 됩니다.

When opening the new content, if the number of open tab contents exceeds the maximum stateful count, the state of the first opened tab content is removed, and clicking on the tab with the removed state, it will reload the content.

값이 0 이면 최대 상태유지 컨텐츠 수를 제한 하지 않습니다.

A value of 0 does not limit the maximum number of stateful content.
maxTabs number 0 X

multi 옵션이 true 인 경우 추가 탭 컨텐츠를 열 때마다 웹 브라우저가 느려지지 않도록 최대 탭 컨텐츠 수를 설정할 수 있습니다.

If the multi option is true, the maximum number of tab contents can be set to prevent the web browser from slowing down whenever additional tab contents are opened.

새로운 컨텐츠를 열때 열려있는 탭 컨텐츠의 개수가 최대 탭 개수를 초과 하면 더이상 탭 컨텐츠를 열 수 없습니다. 새로운 컨텐츠를 열려면 기존 탭 중에 하나를 닫아야 합니다.

When opening new content, if the number of open tab contents exceeds the maximum tabs count, the tab contents can no longer be opened. To open new content, you must close one of the existing tabs.

값이 0 이면 최대 컨텐츠 수를 제한 하지 않습니다.

If the value is 0, the maximum number of contents is not limited.
addLast boolean false X

true로 설정 하면 add 메서드를 호출 했을 때 새로운 탭이 마지막에 추가 됩니다.

If set to true, a new tab is added last when the add method is called.
tabScroll boolean false X

true로 설정 하면 탭들을 마우스 드래그나 터치로 스크롤 할 수 있습니다.

If set to true, tabs can be scrolled by dragging the mouse or touching.
tabScrollCorrection boolean false X

탭 요소에 영향을 주는 스타일(CSS)에 의해 마지막 탭이 잘리거나 여백이 생길 수 있습니다. 이때 tabScrollCorrection 객체의 속성으로 다음 옵션 값들을 조절하여 정상적으로 표시되게 할 수 있습니다.

The style (CSS) affecting the tab element may cause the last tab to be truncated or margins. At this time, you can adjust the following option values ​​as properties of the tabScrollCorrection object to display normally.

  • rightCorrectionPx : 마지막 탭이 잘리거나 여백이 생겼을 때 1씩 증감하여 탭의 모양을 보정 할 수 있는 옵션 입니다.
  • rightCorrectionPx : This option allows you to correct the shape of the tab by increasing or decreasing by 1 when the last tab is cut or a margin is created.
N("#docs").docs({
    tabScrollCorrection : {
		rightCorrectionPx : 1
	}
});

Config(natural.config.js) 의 N.context.attr("ui.shell").docs 프로퍼티에 tabScrollCorrection 옵션을 설정하면 전체 N.docs 컴포넌트에 적용 됩니다.

Setting the tabScrollCorrection option in the N.context.attr("ui.shell").docs property of Config(natural.config.js) applies to all N.docs components.
closeAllRedirectURL string null X

"전체 닫기" 버튼을 클릭 했을 경우 closeAllRedirectURL 옵션 값이 null 이면 활성 화 된 탭을 제외한 나머지 탭들을 닫아고 url 문자열을 입력 하면 해당 url 로 리다이렉트 됩니다.

When the "Close All" button is clicked, if the value of the closeAllRedirectURL option is null, all other tabs except the active tab are closed and if you input the url string, it will be redirect to the url.

SPA(Single Page Application) 로 개발 할 때 홈(메인 인덱스)으로 이동하는 url 을 입력 하여 탭 전체가 닫힐때 만 이라도 페이지 리다이렉션이 일어나서 브라우저의 가비지 리소스들이 정리 되게 하는것을 권장 합니다.

When developing with SPA(Single Page Application), it is recommended to enter the url to go to the home(Main index page) so that even when the entire tab is closed, page redirection occurs, so that the garbage resources of the browser are cleaned.
entireLoadIndicator boolean false X

true로 설정 하면 페이지가 로딩 될 때 실행 되는 Ajax 요청들이 모두 완료 될때 까지 진행 상태 바를 보여 줍니다.

If set to true, the progress bar is displayed until all Ajax requests executed when the page is loaded are completed.
entireLoadScreenBlock boolean false X

true로 설정 하면 페이지가 로딩 될 때 실행 되는 Ajax 요청들이 모두 완료 될때 까지 화면을 차단하여 Double Submit 을 방지 합니다.

If set to true, double submission is prevented by blocking the screen until all Ajax requests executed when the page is loaded are completed.
entireLoadExcludeURLs array [] X

entireLoad 관련 이벤트나 옵션(entireLoadIndicator, entireLoadScreenBlock 등)에서 entireLoadExcludeURLs 로 지정한 URL 들을 캡쳐 대상에서 제외 합니다.

Excludes URLs specified by entireLoadExcludeURLs from the entireLoad(entireLoadIndicator, entireLoadScreenBlock, etc.) related event or option.

페이지가 처음 로딩 될 때 N.docs 컨텍스트 바깥에서 로딩되는 Ajax 호출을 제외하고 싶을 때 설정 바랍니다.

Set this when you want to exclude Ajax calls that are loaded outside of the N.docs context when the page is first loaded.
alwaysOnTop boolean false X

true로 설정하면 메뉴목록 대화상자를 항상 최 상위에 표시 합니다.

If set to true, The menu list dialog is always displayed at the top.
alwaysOnTopCalcTarget string div, span, ul, p, nav, article, section, header, footer, aside X

alwaysOnTop 옵션 적용 시 최 상위 z-index를 계산하기 위한 대상 요소들을 지정 합니다.

When applying the alwaysOnTop option, specify the target elements for calculating the highest z-index.

jQuery selector 구문으로 지정 합니다.

It is specified using jQuery selector syntax.

N.docs 관련 요소들이 다른 요소에 의해 가려 질 때 가리고 있는 요소 셀렉터를 추가 해 주세요.

Add an element selector that is obscured when elements related to N.docs are obscured by other elements.
onBeforeLoad function null X

컨텐츠가 로딩 되기 전 실행 되는 이벤트 입니다.

This event is executed before the content is loaded.

onBeforeLoad : function(docId, target) {
	// docId : Document Id
	// target : Elements to be loaded content

	// get document info
	var doc = this.doc(docId);
}

열린 페이지가 닫히기 전 까지 다시 호출 되지 않습니다.

It is not called again until the opened page is closed.

페이지가 로딩되기 전에 실행 되므로 cont 메서드로 불러온 페이지의 Controller 를 가져 올 수 없습니다.

It is executed before the page is loaded, so you cannot get the Controller object of the page called by the cont method.
onLoad function null X

페이지가 로딩 된 후 실행 되는 이벤트 입니다.

This event is executed after the content is loaded.

onLoad : function(docId) {
	// docId : Document Id

	// get document info
	var doc = this.doc(docId);
	// get document's Controller
	var cont = this.cont(docId);
	var view = cont.view;
	var request = cont.request;
}

열린 페이지가 닫히기 전 까지 다시 호출 되지 않습니다.

It is not called again until the opened page is closed.
onBeforeEntireLoad function null X

페이지가 로딩 될 때 실행 되는 Ajax 요청들이 캡쳐 되기 전에 실행되는 이벤트 입니다.

This event fires before the Ajax request is captured when the page is loaded.

onBeforeEntireLoad : function(docId) {
	// docId : Document Id

	// get document info
	var doc = this.doc(docId);
}

컨텐츠 로딩 전 최초 한번은 컨텐츠가 로딩되기 전에 이벤트가 실행 되므로 cont 메서드로 불러온 페이지의 Controller object 를 가져 올 수 없습니다.

The first time before loading the content, the event is fired before the content is loaded, so you cannot get the Controller object of the page called by the cont method.
onErrorEntireLoad function null X

페이지가 로딩 될 때 실행 되는 Ajax 요청들이 모두 완료되기 전에 에러가 발생 되었을때 실행되는 이벤트 입니다.

This event fires when an error occurs before all Ajax requests complete when the page loads.

onErrorEntireLoad : function(e, request, xhr, textStatus, callback) {
	// e(arguments[0]) : ErrorThrown
    // request(arguments[1]) : Communicator.request
    // xhr(arguments[2]) : jQuery XMLHTTORequest
    // textStatus(arguments[3]) : "success"(submit 콜백에서 에러 발생 시) 또는 "error"(서버에서 오류 발생 시)
    // callback(arguments[4]) : textStatus 값이 "success" 일때 submit 메서드의 인자로 지정한 콜백 함수
}

onErrorEntireLoad의 이벤트 핸들러 함수는 N.comm.error 메서드의 callback 과 같습니다.

The event handler function of onErrorEntireLoad is the same as the callback of the N.comm.error method.
onEntireLoad function null X

페이지가 로딩 될 때 실행 되는 Ajax 요청들이 모두 완료된 후에 실행되는 이벤트 입니다.

This event fires after all Ajax requests complete when the page loads.

onEntireLoad : function(docId) {
	// docId : Document Id

	// get document info
	var doc = this.doc(docId);
	// get document's Controller
	var cont = this.cont(docId);
	var view = cont.view;
	var request = cont.request;
}
