예제 Examples

1. N.form 이나 N.grid 로 데이터를 바인드 하기 전에 컬럼 값이 바이드 될 요소의 data-format 속성에 포멧 룰을 선언 하기.

1. Let's declare a format rule in the data-format attribute of the element to which the column value will be bound before binding data with N.form or N.grid.

<input id="startDate" type="text" data-format='[["date", 8]]'>

위와 같이 입력요소에 포멧 룰을 선언하면 입력 요소에 focus-in 되면 포멧이 해제 되고 focus-out 되면 다시 포멧팅 됩니다.

When the format rule is declared on the input element as above, unformatted when focus-in to the input element and reformatted when focus-out from the input element.

<span id="startDate" data-format='[["date", 8]]'></span>

위와 같이 입력요소가 아닌 요소에 포멧 룰을 선언하면 포멧 된 값이 요소의 text 로 입력 됩니다.

If a format rule is declared on an element that is not an input element as above, the formatted value is put as the text of the element.

N.grid 나 N.form 의 data 매서드를 호출하여 컴포넌트의 데이터를 가져와 보면 포멧팅 되지 않은 값으로 세팅 되어 있을 것 입니다. 요소에 보이는 값들만 포멧팅 하기 때문에 서버에 데이터를 전송 할 때 unforamt 하지 않아도 됩니다.

If you call the data method of N.grid or N.form to get the component data, it will be set to the unformatted value. It only formats the values ​​shown in the element, so there is no need to unformat when sending data to the server.

2. N.formatter 라이브러리로 포멧팅 하기.

2. Let's format it with the N.formatter library.

요소에 포멧 룰을 선언 할 때 data-format='[["date", 8]]' 속성 값에서 "date" 가 포멧팅룰 이고 8 이후 부터는 포멧 파라미터 이고 포멧팅 할 문자열은 입력 요소의 value 값이나 그외 요소의 text 입니다.

When declaring a format rule on an element, in data-format='[["date", 8]]' attribute value, "date" is the formatting rule, and after 8 is the format parameters, and the string to be formatted is the value of the input element or the text of other element.

이 선언을 명령으로 처리 할 때는 다음와 같이 사용 할 수 있습니다.

When processing this declaration as a imperative, you can use it as follows:

// N.formatter.rule("string to format", [arg[0], arg[1], arg[n]]);
N.formatter.date("20201011", [8]); // 2020-10-11

3. 요소에 바인딩 된 Formatter 관련 이벤트를 직접 실행 하기.

3. Let's excute Formatter related events directly bound to elements.

Formatter 는 룰 대신 입력요소를 지정하면 입력 요소에 "format" 이벤트와 "unformat" 이벤트를 생성 합니다. 이 이벤트들은 jQuery trigger 메서드로 직접 실행 할 수 있습니다.

Formatter creates "format" and "unformat" events to input elements when an input element is specified instead of a rule. These events can be executed directly with the jQuery trigger method.

N("input[name='formatEl']").trigger("unformat");
N("input[name='formatEl']").trigger("format");

4. 동적으로 포멧룰을 변경하고 바로 반영 하기

4. Let's change the formatting rules dynamically and apply them immediately.

아래 코드가 작동 하려면 컴포넌트로 데이터를 바인드 하기 전에 대상 요소에 data-format 속성으로 포멧룰이 1개 이상 선언 되어 있어야 합니다.

For the code below to work, at least one format rule must be declared as a data-format attribute in the target element before binding data to the component.

N("selector").data("format", [["trimtoempty"]]).trigger("format");

전체 포멧팅 룰에 대한 설명은 [포멧 룰 목록] 탭을 참고 하세요.

For a description of the entire formatting rule, see the [Format rule list] tab.
