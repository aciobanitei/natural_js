예제 Examples

1. N.form 이나 N.grid 로 데이터를 바인드 하기 전에 컬럼 값이 바이드 될 요소의 data-validate 속성에 유효성 검증 룰을 선언 하기.

1. Let's declare a validate rule in the data-validate attribute of the element to which the column value will be bound before binding data with N.form or N.grid.

<input id="col01" type="text" data-validate='[["required"]]'>

위와 같이 입력요소에 선언 하면 해당 요소에서 포커스 아웃 되면 유효성 검증이 실행 되고 유효성 검증 실패 시 실패 원인을 입력 요소 옆에 툴팁으로 보여 줍니다.

If declared in the input element as above, validation is executed when the element is focus-out, and when validation fails, the cause of the failure is shown as a tooltip next to the input element.

2. N.validator 라이브러리로 유효성 검증 하기.

2. Let's validate it with the N.validator library.

요소에 유효성 검증 룰을 선언 할 때 data-validate='[["maxbyte", 8]]' 속성 값에서 "date" 가 유효성 검증 룰 이고 8 이후 부터는 유효성 검증 파라미터 이고 유효성 검증 할 문자열은 입력 요소의 value 값이나 그외 요소의 text 입니다.

When declaring a validation rule on an element, in data-validate='[["maxbyte", 8]]' attribute value, "maxbyte" is the validation rule, and after 8 is the validate parameters, and the string to be validated is the value of the input element.

이 선언을 명령으로 처리 할 때는 다음과 같이 사용하면 됩니다.

When processing this declaration as a imperative, you can use it as follows:

N.validator.rule("string to validate", [arg[0], arg[1], arg[n]]);
N.validator.maxbyte("string to format", [8]); // false

3. 요소에 바인딩 된 Validator 관련 이벤트를 직접 실행 하기.

3. Let's excute Validator related events directly bound to elements.

Validator 는 룰 대신 입력요소를 지정하면 입력 요소에 "validate" 이벤트를 생성 합니다. 이 이벤트는 jQuery trigger 메서드로 직접 실행 할 수 있습니다.

Validator creates "validate" event to input elements when an input element is specified instead of a rule. These event can be executed directly with the jQuery trigger method.

N("input[name='validatetEl']").trigger("validate");

4. 동적으로 유효성 검증룰을 변경하고 바로 반영 하기

4. Let's change the validation rules dynamically and apply them immediately.

아래 코드가 작동 하려면 컴포넌트로 데이터를 바인드 하기 전에 대상 요소에 data-validate 속성으로 유효성 검증룰이 1개 이상 선언 되어 있어야 합니다.

For the code below to work, at least one validation rule must be declared as a data-validate attribute in the target element before binding data to the component.

N("selector").data("validate", [["required"]]).trigger("validate");

전체 유효성 검증 룰에 대한 설명은 [유효성 검증 룰 목록] 탭을 참고 하세요.

For a description of all validation rules, refer to the [Validation Rule List] tab.
