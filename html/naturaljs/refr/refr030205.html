예제 Examples

1. N.form 이나 N.grid, N.list 초기화 전에 값이 바인드 되는 컬럼 태그에 data-validate 속성에 정의 하여 선언형으로 처리

<input id="col01" type="text" data-validate='[["required"]]'>

위와 같이 입력요소에 정의하면 해당 요소에서 포커스 아웃 되면 검증이 실행 되고 검증 실패 시 실패 원인을 입력 요소 옆에 툴팁으로 보여 줍니다.

2. 처리 함수를 직접 호출하여 검증을 명령형으로 처리

검증을 선언형으로 처리 할 때 data-validate='[["maxbyte", 8]]' 구문 에서
"검증 할 문자" 는 사용자가 입력한 문자가 자동으로 입력 되고
"maxbyte" 가 검증룰 이고 8 은 인자(arguments) 입니다.

이런 구문을 명령형으로 처리 할 때는 다음과 같이 사용하면 됩니다.

N.formatter.검증룰("검증 할 문자", [arg[0], arg[1], arg[n]]);
// 한글이름
N.validator.maxbyte("가나다라마", [8]); //--> false

3. 요소에 바인딩 된 Validator 이벤트를 직접 실행

Validator 는 룰 대신 입력요소를 지정하면 입력 요소들 마다 "validate" 이벤트를 바인드(bind) 해 줍니다.
Validator 가 적용 된 입력요소를 jQuery selector로 가져와 jQuery trigger 메서드로 이벤트를 실행하면 입력된 값을 명령으로 validate 할 수 있습니다.

N("input[name='validatetEx']").trigger("validate");

4. 동적으로 검증룰을 변경하고 바로 검증

N.form 이나 N.grid 등에서 사용 할 때는 아래 코드가 작동 하기 위해선 대상 요소에 data-validate 속성으로 검증룰이 1개 이상 선언 되어 있어야 합니다.

N("selector").data("validate", [["required"]]).trigger("validate");

전체 검증 룰 들은 Validation Rules 탭을 참고 하세요.
